Polecenia:

1. gita miałem już zainstalowanego, ale użyłbym polecenia "sudo apt install git" gdyby tak nie było
2. Sklonowałem repozytorium za pomocą odpowiedniego polecenia
   git clone - kopiuje zawartość wskazanego zdalnego repozytorium na lokalną maszynę, tworzy wszystkie branche obecne w sklonowanym repozytorium, ustawia je jako śledzące swoje zdalne odpowiedniki oraz przełącza na główny branch sklonowanego repozytorium
4. Stworzyłem nowy branch, utworzyłem na nim odpowiednie foldery. Umieściłem w nich niniejsze sprawozdanie, a następnie zacommitowałem je na swojego brancha
   git status - wyświetla status obecnego folderu roboczego - wszystkie zmienione, utworzone lub usunięte pliki oraz zawartość staging area
   git checkout - pozwala na przełaczanie się między branchami (i nie tylko). Użyta opcja -b powoduje, że przed przełączeniem git tworzy nowy branch o podanej nazwie
   git add - dodaje pliki do staging area
   git commit - tworzy nowy commit. Opcja -m pozawala na edycje wiadomości commita prosto z linii polecenia, bez otwierania edytora tekstu
6. Utworzyłem zrzuty ekranu i je zacommitowałem. Następnie zmieniłem nazwę każdemu z nich i również zacommitowałem te zmiany.
7. Wykonałem git fetch brancha origin/master i zmergowałem go do mojego brancha MA306420. Następnie przełączyłem się na branch master i wykonałem git pull, a następnie git merge na mojej gałęzi. Po zakończeniu merge spushowałem powstały merge commit na brancha master.
   git fetch - pobiera zmiany z remote, ale pozostawia je na osobnym branchu bez wykonywania merge do aktualnego brancha
   git pull - odpowiednik wykonania git fetch && git merge origin/<branchname>
   git merge - integruje zmiany z podanego brancha do aktualnego brancha
