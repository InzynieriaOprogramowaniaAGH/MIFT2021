1. Na komputerze miałam zainstalowanego już klienta, konkretnie GIT BASH.

2. Użyłam operacji git clone link do repo, żeby sklonować repozytorium. Po tym prowadzący powiedział,żeby zrobić git pull więc zrobiłam git pull. W commitowanych plikach jest także zdjęcie o nazwie graf, które przedstawia prowizoryczny rysunek grafu zależności. 

3. Zapoznałam się z dokumentem. 

4. Stworzyłam nowy branch poleceniem git checkout -b nazwa, co spowodowało automatyczne przejście na tego brancha, a tam poleceniem mkdir utworzyłam odpowiednie katalogi. 

5. W katalogu lab01 poleceniem touch utworzyłam plik sprawozdanie.txt(tj. ten plik). Poleceniem git add plik dodałam plik do repo, a następnie poleceniem git commit -m plik commitowałam go. 

6. Następnie po dokonaniu zmian w sprawozdaniu oraz zrobieniu skrinów dodałam wszystkie potrzebne pliki dodałam do repo(git add .), a później commitowałam je za pomocą git commit. Później pushowałam poleceniem git push. 

7. Na koniec zmergowałam mastera z moim branchem poleceniem git merge nazwa. Później próbowałam pullować, ale niestety wyskakiwał błąd więc użyłam komendy git stash. Następnie git pull(z sukcesem). Na koniec git push. Ponieważ w trakcie pracy pojawiały się pewne problemy, dodaje do repozytorium także skrin historii z basha, na którym widoczne są wszystkie użyte przeze mnie komendy.
