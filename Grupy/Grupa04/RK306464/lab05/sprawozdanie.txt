1. Stworzyłem i uruchomiłem kontener z Jenkinsem zgodnie z instrukcją rozpoczynając od 		- utworzenia bridga poleceniem docker network create jenkins
	następnie komendą 
		docker run \
		  --name jenkins-docker \
		  --rm \
		  --detach \
		  --privileged \
		  --network jenkins \
		  --network-alias docker \
		  --env DOCKER_TLS_CERTDIR=/certs \
		  --volume jenkins-docker-certs:/certs/client \
		  --volume jenkins-data:/var/jenkins_home \
		  --publish 2376:2376 \
		  docker:dind \
		  --storage-driver overlay2 
	uruchomiłem obraz docker dind co umożliwiało wykonywanie poleń dockera w jenkinsie
	
	- Utworzyłem Dockerfile z jenkinsem zgodnie z instrukcją na stronie, natomiast usunąłem 		linię USER Jenkins, która powodowała problemy z permission denied
	- Poleceniem docker build -t myjenkins-blueocean: . zbudowałem obraz dockerowy
	- A przy pomocy polecenia
	docker run \
	  --name jenkins-blueocean \
	  --rm \
	  --detach \
	  --network jenkins \
	  --env DOCKER_HOST=tcp://docker:2376 \
	  --env DOCKER_CERT_PATH=/certs/client \
	  --env DOCKER_TLS_VERIFY=1 \
	  --publish 8080:8080 \
	  --publish 50000:50000 \
	  --volume jenkins-data:/var/jenkins_home \
	  --volume jenkins-docker-certs:/certs/client:ro \
	  myjenkins-blueocean
	  Uruchomiłem jenkinsa na localhoscie.
