1. Zarejestrowano w repozytorium webhook'a oraz zainstalowano ngrok, który służy do tunelowania localhosta.
   $ sudo apt install snapd
   $ sudo snap install ngrok
   Ngrok został uruchomiony za pomocą poniższej komendy:
   $ ngrok http 8080
   Otworzyła ona port na którym jest Jenkins.

2. Sprawdzono i udokumentowano automatyczne wyzwalanie pipeline

3. Jenkins file został rozszerzony o nowy stage("Deploy")
	stage('Deploy') {
			steps {
				echo 'Deploy'
				sh 'docker build -t deploy -f Dockerfile_socketio_deploy .'
			}
			post {
				always{
					echo 'Finished'
				}
				failure{
					messageFunction('DEPLOY', 'Failure')
				}
				success{
					messageFunction('DEPLOY', 'Success')
				}
            		}
		}

4. Deploy jest wykonywany na dedykowany kontener dockerowy:
	sh 'docker build -t deploy -f Dockerfile_socketio_deploy .'
   Użytkownik jest powiadamiany o sukcesie oraz niepowodzeniu etapu wdrożenia
	messageFunction('DEPLOY', 'Failure'), messageFunction('DEPLOY', 'Success')
	
   Dockerfile został dodany do repozytorium. Jego zawartość jest następująca:
    FROM ubuntu:latest
    ENV TZ=Europe/Warsaw
    RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone
    RUN apt-get -y update
    RUN apt-get install git -y
    RUN apt-get install npm -y
    COPY . Socket.io-Messenger/
    WORKDIR Socket.io-Messenger

5. Niestety po mimo wielu prób nie udało się uruchomić poprawnie stage("Deploy").
