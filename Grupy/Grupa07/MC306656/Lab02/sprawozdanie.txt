Do utworzenia pary kluczy prywatny/publiczny używamy komendy ssh-keygen -t ed25519 -C po której wpisujemy maila, na którym zarejestrowane jest nasze konto GitHub. Następnie poprzez komendę eval 'ssh-agent -s' tworzymy działającego agenta SSH do obsługi klucza SSH. Kolejnym krokiem jest dodanie klucza do konta GitHub. Komenda clip zaznacza nasz plik z kluczem publicznym, który to wklejamy w ustawieniach naszego konta. Dzięki temu po użyciu komend git remote -v, git remote set-url origin git@github.com:InzynieriaOprogramowaniaAGH/MIFT2021.git" (zmienia połącznie na SSH), git remote -v nasz git zostaje poinformowany o zmianie sposobu logowania. W kolejnym zadaniu poprzez użycie komendy git tag -a nadajemy tag na aktualnego commita, następnie komendą git push --follow-tags nadajemy na origin. W kolejnym kroku należało przenieść się na istniejący branch swojej grupy i od niego stworzyć branch, na którym należało utworzyć pre-commit weryfikujący czy każdy commit zaczyna się od MC306656. Do tego użyłam komendy chmod +x commit-msg, którą wykonałam po utworzeniu pliku commit-msg w foldere .git/hooks. Ostatecznie zrobiłam merga na brancha grupowego i wykonałam pull requesta.
