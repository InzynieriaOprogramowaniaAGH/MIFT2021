1. Wykorzystano kontener z poprzednich zajęć wraz z DockerHubem.

2. Zainstalowano zgodnie z instrukcją Kubernetes oraz Minikube.

    $ curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
    $ sudo install minikube-linux-amd64 /usr/local/bin/minikube

3. Uruchomiono Minikube Dashboard

    $ minikube dashboard

3.1 Możliwości Minikube Dashboard:

    - kontrola deployów i jobów
    - kontrolowanie podsów
    - edycja, sterowanie konterami
    - monitorowanie zasobów (CPU, memory)
    - zestawianie danych/statystyki
    - sterowanie zmiennymi środowiskowymi
    - wygoda w przeglądaniu logów

4. Przyznano alias dla kubectl, by wyeliminować problemy z widocznością flag oraz uruchomiono kontener z komunikatorem w Kubernetes:

    $ alias kubectl="minikube kubectl --"
    $ kubectl run deltachat --image=krzysztofgac/myjenkins-blueocean --port=9999 --labels app=deltachat

4.1 Ustawiono porty

    $ kubectl port-forward deploy/deltachat 8080:9999

5. Kontener został pomyślnie uruchomiony w klastrze (zrzut lab11_5.png)
