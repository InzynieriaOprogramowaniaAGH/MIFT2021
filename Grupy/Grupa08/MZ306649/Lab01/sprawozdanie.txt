Wykorzystywanie polecenia

git clone - polecenie klonuje istniejące repozytroium do wybranego katalogu

git branch <nazwa> - polecenie tworzy nowy branch o podanej nazwie

git checkout <nazwa brancha> - polecenie przłącza na podany branch

mkdir <nazwa> - utworzenie katalogu o podanej nazwie

touch <nazwa> - stworzenie pliku o podanej nazwie

git add <nazwa> - dodanie do repozytorium pliku o podanej nazwie

git commit - polecenie zapisuje zmiany w repozytorium (-m "komentarz commita")

git merge <nazwa brancha> - polecenie łączy historię podanego brancha do brancha na którym użytkownik wykonuje polecenie

git push - polecenie przesyła pliki z lokalnego repozytorum do repozytorium zdalnego

Przebieg ćwiczenia
1. Klient Git był już wcześniej zainstalowany na komputerze.
2. W wybranym folderze uruchomiono Git bash i wykonano polecenie git clone z podanym linkiem.
3. Zapoznano się z dokumentem tekstowym zawierającym instrukcjęj do zajęć.
4. Utworzono nowy branch poleceniem git branch MZ306640, przełączono się na utworzony branch poleceniem git checkout MZ306649, utworzono wymagane katalogi oraz pliki.
5. Uzupełniono plik ze sprawozdaniem oraz dodano go do repozytorium poleceniami git add oraz git commit
6. Dodano do repozytorium zrzut ekranu z zdania 4.
7. Przełączono się na branch master oraz wykonano merge poleceniem git merge MZ306649. Wykonano commit i push na remot'a.
