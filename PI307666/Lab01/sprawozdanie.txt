1. Sklonowano repozytorium na dysk komputera - git clone.
2. Utworzono nowy branch i automatycznie przełączono się na nową gałąź - git checkout -b PI307666
3. Utworzono folder - polecenie mkdir PI307666
4. Utworzono plik z sprawzodaniem - touch sprawozdanie.txt
5. Dodano plik do repozytorium gita - git add sprawozdanie.txt
6. Wykonano commit z dodanymi plikami - git commit -m "wiadomosc"
7. Dodano zrzuty ekranów - git add *.png
8. Sprawdzono pliki do wykonania commit - git status
9. Zmieniono nazwy plików - gir mv staranazwa nowanazwa
10. Przełączono na branch master - git checkout master
11. Zaktualizowano repo - git pull origin master
12. Wykonano łączenie zmian z repozytoriów (merge) - git merge PI307666
13. Wypchnięto zmiany na repozytorium GitHub - git push origin master.
