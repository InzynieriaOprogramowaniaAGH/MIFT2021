Na początku zostało wykonane polecenie git clone, za pomocą którego zostało "sklonowane" repozytorium na repozytorium lokalne.

git branch - polecenie do utworzenia nowego brancha,
git checkout - przejście na danego brancha, możliwe było użycie flagi "-p" za pomocę którego można odrazu stworzyć brancha oraz odrazu na niego przejść.

git add "plik" - nazwa pliku, który zostaje dodany do staging area
git add . - dodanie wszystkich nowych plików do staging area

git commit - wykonanie commita na brancha, gdzie się znajdujemy
git commit -m - nadanie odrazu temu commitowi nazwy

można użyć git commit -am - jest to dodanie do staging area i odrazu commit z nadaniem nazwy

git status - z pomoca niego można sprawdzić gdzie znajduje się dany plik: staging area/repozytorium.
